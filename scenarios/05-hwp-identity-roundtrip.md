# Scenario 05 — HWP Binary-Identity Round-trip

**Persona:** 호환성 (compatibility)
**Sprint:** 1.5 — Binary-Identity Save Gate
**Goal:** Prove that a `.hwp` file opened and re-saved by
`rhwp-mcp-server` survives a structural identity check — the core
guarantee Form-Filling customers need before migrating workflows off
the Hancom desktop client.

## Story

> 인사팀 김 대리: "정부24에서 받은 .hwp 양식을 우리 자동화에 넣고 싶은데,
> 다시 저장한 파일을 한컴오피스로 열었을 때 깨지면 곤란해요. 진짜
> '바이트 단위로 똑같이' 안 돼도, 적어도 구조가 무너지지 않는다는 증거가
> 필요해요."

This scenario gives 김 대리 the evidence she asked for: a reproducible
round-trip that the gate runner records as PASS, with the per-case
verify reports she can attach to her vendor evaluation.

## Walkthrough

### 1. Materialize a corpus

The scenario uses the synthetic corpus generated by the project so
김 대리 can reproduce the result without sharing internal documents:

```sh
$ npm run corpus:generate
> rhwp-mcp-server@0.1.0-alpha.0 corpus:generate
> tsx scripts/generate-synthetic-corpus.ts

wasm warm: 64ms (rhwp/core v0.7.13, 5062315 bytes)
  blank              → corpus/synthetic/blank.hwp (12800 bytes)
  text-only          → corpus/synthetic/text-only.hwp (12800 bytes)
  table-only         → corpus/synthetic/table-only.hwp (13312 bytes)
  paragraph-style    → corpus/synthetic/paragraph-style.hwp (12800 bytes)
  mixed              → corpus/synthetic/mixed.hwp (13312 bytes)

Wrote 5 synthetic cases + manifest.json
```

If 김 대리 has her own `.hwp` files, she can drop them into
`corpus/private/` (gitignored — they never leave her machine) and the
runner picks them up automatically.

### 2. Run the gate

```sh
$ npm run gate:binary-identity
> rhwp-mcp-server@0.1.0-alpha.0 gate:binary-identity
> tsx scripts/corpus-runner.ts

wasm warm: 32ms (rhwp/core v0.7.13, 5062315 bytes)

Binary-Identity Save Gate (Sprint 1.5) — rhwp 0.7.13
  Cases : total=5  pass=5  fail=0  skip=0
  Rate  : 100.0%  (Wilson 95% [56.6%, 100.0%])
  Gate  : PASS (threshold ≥90%)

  [PASS] synthetic/blank             (pages 1, bytes 12800→12800)
  [PASS] synthetic/mixed             (pages 1, bytes 13312→13312)
  [PASS] synthetic/paragraph-style   (pages 1, bytes 12800→12800)
  [PASS] synthetic/table-only        (pages 1, bytes 13312→13312)
  [PASS] synthetic/text-only         (pages 1, bytes 12800→12800)
```

Exit code: `0`. The runner also writes `corpus-report.json` with the
full per-case `exportHwpVerify()` reports for audit.

### 3. Inspect the evidence

```sh
$ node -e "const r=require('./corpus-report.json'); for (const c of r.results) \
  console.log(c.caseId, '  strict='+c.strictByteEqual, '  verifyBefore=', c.verifyBefore);"
synthetic/blank           strict=true  verifyBefore= { bytesLen: 12800, pageCountBefore: 1, pageCountAfter: 1, recovered: true }
synthetic/mixed           strict=true  verifyBefore= { bytesLen: 13312, pageCountBefore: 1, pageCountAfter: 1, recovered: true }
synthetic/paragraph-style strict=true  verifyBefore= { bytesLen: 12800, pageCountBefore: 1, pageCountAfter: 1, recovered: true }
synthetic/table-only      strict=true  verifyBefore= { bytesLen: 13312, pageCountBefore: 1, pageCountAfter: 1, recovered: true }
synthetic/text-only       strict=true  verifyBefore= { bytesLen: 12800, pageCountBefore: 1, pageCountAfter: 1, recovered: true }
```

Every case round-tripped:

- `recovered: true` — rhwp re-constructed the document IR without loss.
- `pageCountBefore === pageCountAfter` — no page drift.
- `strict=true` — byte streams matched exactly.

The last point is the strongest possible evidence: 김 대리 can hand this
to her own QA team verbatim.

## Acceptance for this scenario

- The runner exits 0.
- Pass rate ≥ 90% (Wilson 95% interval recorded).
- At least one case in the corpus.
- `corpus-report.json` exists and contains a per-case verify report.

## Limits

- The Sprint 1.5 corpus is synthetic; real-world `.hwp` files may
  exhibit different behavior. The runner accepts real files in
  `corpus/forms/` and `corpus/private/` — see ADR-0002 §"Fallback
  policy" for what happens if the gate flips to FAIL on a wider corpus.
- "Structural identity" is the v0.1.5 guarantee. Per-OLE2-stream
  binary diff lands in v0.2 (ADR-0002 §"v0.2 — per-stream byte diff").
