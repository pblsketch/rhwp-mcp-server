## Context

The locked plan (`Sprint 1.5 — Binary-Identity Save Gate`) demands a
quantitative check on `@rhwp/core`'s `.hwp` (HWP 5.0 binary) write path
*independent of* the Field API stability gate validated in Sprint 1. The
rationale: a Form-Filling product that can read `.hwp` but cannot save
back to `.hwp` would force every customer to migrate to `.hwpx`, which
contradicts the 호환성 vector of the spec.

The gate threshold is **≥ 90% binary-identity match** across the `.hwp`
inputs, with a Wilson 95% confidence interval recorded for posterity.

## Decision

### v0.1.5 — Adopt the Pass B definition implemented in `scripts/corpus-runner.ts`

A case is considered "identity-preserved" when **all** of:

1. `new HwpDocument(inputBytes)` succeeds.
2. The first `exportHwpVerify()` reports `recovered: true` and
   `pageCountBefore === pageCountAfter`.
3. `exportHwp()` returns a non-empty `Uint8Array`.
4. Re-opening that re-exported byte stream succeeds.
5. The second `exportHwpVerify()` again reports `recovered: true` and the
   page count matches the pre-export count.

We additionally compute a **strict byte-equal** flag and a
**whitelist-matched** flag per case for visibility; only criteria 1–5
are load-bearing for the gate.

### Baseline measurement (Sprint 1.5, this ADR)

- Corpus: 5 synthetic cases generated by
  `scripts/generate-synthetic-corpus.ts` (blank, text-only, table-only,
  paragraph-style, mixed).
- Result: **5 / 5 pass**, **100.0%**, Wilson 95% CI [56.6%, 100.0%].
- Bonus signal: **5 / 5 strict byte-equal** — rhwp 0.7.13 produces a
  deterministic byte stream for the synthetic inputs (no wall-clock or
  UUID drift observed).
- Verdict: **Gate 1.5 — PASS**.

Per the locked plan, this clears Sprint 2 to remain in scope. Sprint 2
authoring tools have already shipped (`feat(sprint-2)` at commit
`eadffc3`).

### Fallback policy — what happens if a future measurement falls below threshold

If a subsequent rhwp release, or expansion of the corpus with
`corpus/forms/` and `corpus/private/`, drops the pass rate below **90%**:

1. File minimal-repro issues upstream at
   `https://github.com/edwardkim/rhwp/issues`. Mandatory before any
   downstream descope. Include the failing `corpus-report.json` entry
   and the offending bytes (where licensing permits).
2. Evaluate two descope options and pick one within 1 calendar week:
   - **Option α — HWPX-only output.** Document `hwp_save_as` as
     `.hwpx`-only when reading a `.hwp` source. The MCP tool schema is
     unchanged (output is still a path), but the README and tool
     description gain a "best-effort .hwp save" marker. Form-Filling
     customers retain `.hwpx` round-trip.
   - **Option β — Pure Option D.** Drop Form-Filling from v0.1.0 and
     pivot to the "Render + Round-trip Smoke First" path from the
     plan's rejected alternatives list. Higher cost, used only if
     Option α would invalidate the spec's 호환성 vector.
3. Record the chosen option as ADR-0002b (amendment) with the failing
   measurement attached.

### v0.2 — per-stream byte diff (deferred)

The v0.1.5 gate accepts a successful re-import as a whitelist match. The
v0.2 roadmap upgrades this to a per-OLE2-stream binary diff that
excludes the streams enumerated in `corpus/identity-whitelist.json`
(timestamps, generator string, etc.) and reports the residual.

Target: <5% residual diff per case before claiming "bit-level identity"
in marketing material. Until v0.2 ships, public-facing copy must say
"structural identity" or "round-trip safe", never "byte-identical".

## Consequences

### Positive

- Sprint 2 was unblocked by the measured baseline; the locked plan's
  staged delivery proceeds on schedule.
- The gate is **reproducible from a fresh clone** — no external corpus
  required to exercise it. Real-world signal can be added incrementally
  via `corpus/forms/` and `corpus/private/`.
- The 100% pass rate on synthetic cases gives us a *signal-amplifying*
  baseline: any real-world case that fails will stand out as a true
  upstream defect, not a flaky measurement.

### Negative

- The Wilson lower bound (56.6%) on N=5 is not by itself proof of
  production readiness. The gate is *necessary*, not sufficient. The
  Sprint 3 N=30 corpus closes this gap.
- "Strict byte-equal" on synthetic cases may not generalize — real
  `.hwp` files carry DocInfo timestamps that synthetic cases skip.
  Treat `strictByteEqual=true` as a *bonus*, not the contract.

### Neutral

- ADR-0001 (image renderer) remains independent. Sprint 3 owns that
  decision; ADR-0002 does not depend on it.
