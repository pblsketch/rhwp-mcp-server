# corpus/

This directory holds the `.hwp` and `.hwpx` test corpora used by the
Sprint 1 / Sprint 1.5 / Sprint 3 quality gates.

## Layout

```
corpus/
├── synthetic/         # generated by `npm run corpus:generate` (committed)
├── forms/             # curated public-domain forms (committed; see SOURCES.md)
├── private/           # user's local corpus — GITIGNORED
├── identity-whitelist.json
├── SOURCES.md
└── README.md          # this file
```

### `synthetic/` — generated, committed

Deterministic outputs of `scripts/generate-synthetic-corpus.ts`. The cases
cover blank, text-only, table-only, paragraph-style, and mixed
constructions. Regenerate with:

```sh
npm run corpus:generate
```

These files exist so CI (and a fresh `git clone`) can run the Binary-
Identity gate without any external dependency.

### `forms/` — public corpus, committed

Hand-curated real-world forms (정부24, school templates, etc.). Every entry
MUST be cleared for inclusion via `SOURCES.md` (license, provenance,
PII status).

### `private/` — local-only, gitignored

Drop your own `.hwp` / `.hwpx` files here. The corpus runner picks them up
automatically. They will never be committed (see `corpus/private/.gitignore`).

This is the recommended path for measuring real-world identity rates on a
sample that contains PII or organization-confidential material.

## Quality gates that consume this directory

| Gate                              | Pass criterion                        | Script                                    |
| --------------------------------- | ------------------------------------- | ----------------------------------------- |
| Sprint 1 — Field round-trip       | ≥ 80% of `.hwpx` cases                | `npm run gate:field-roundtrip` (Sprint 1) |
| Sprint 1.5 — Binary identity save | ≥ 90% of `.hwp` cases                 | `npm run gate:binary-identity`            |
| Sprint 3 — Preview corpus N=30    | ≥ 95% renderable without WASM panic   | `npm run gate:preview` (Sprint 3)         |

Threshold rationales are documented in:

- `docs/decisions/0002-binary-save-fallback.md`
- `docs/measurements/binary-identity-results.md`

## Identity-whitelist

`identity-whitelist.json` documents the rhwp-regenerated metadata fields
that legitimately differ between two sequential saves (timestamps, generator
strings, etc.). The v0.1.5 gate treats any successful re-import of our own
export as a whitelist match; per-field binary diffing is a v0.2 deliverable
tracked in ADR-0002.
